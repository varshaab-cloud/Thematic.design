## Token architecture

Thematic uses a three-tier token system. Base tokens are raw primitives — never referenced in product code. Alias tokens are semantic mappings named by purpose, not value. Component tokens are scoped overrides for when a component needs to deviate from the alias defaults.

## Accessibility

All components are being reviewed against WCAG 2.1 Level AA. Each component doc includes keyboard interaction patterns, ARIA requirements, and screen reader behaviour. The system uses Radix UI primitives for interactive components, which provides correct focus management, keyboard navigation, and ARIA semantics out of the box.

## How to use this documentation

Each component page follows a consistent structure: a live canvas showing the default state, a description of when and how to use the component, individual canvases for each variant or state, and — where the behaviour is non-obvious — a usage guidelines section covering timing, copy rules, and interaction patterns.

The Foundation section covers the raw building blocks: colour palettes, type scale, spacing, elevation, iconography, and the full token reference. The Pages and Wireframes sections show how components compose into real product layouts.
