## Active states

The active prop on a nav item marks the current location. It sets the blue-800 text + blue-100 background treatment. Only one item should ever be active at a time.

    {["State", "Visual", "When to apply"].map(h => <span key={h} style={{ fontSize: 11, fontWeight: 600, color: "#909090", textTransform: "uppercase", letterSpacing: "0.06em" }}>{h}</span>)}
  </div>
  {[
    { state: "active: true", visual: "Blue-800 text + blue-100 background fill", when: "The current route exactly matches this item's destination. Set via router.pathname === item.href." },

    { state: "Hover", visual: "Gray-100 background, gray-900 text — handled by component", when: "Pointer over item. Never override this manually." },
    { state: "Disabled", visual: "50% opacity — use sparingly", when: "Feature not available at this subscription tier. Always show a tooltip explaining why." },
  ].map(({ state, visual, when }, i) => (
    <div key={state} style={{ display: "grid", gridTemplateColumns: "160px 1fr 1fr", padding: "12px 20px", borderBottom: i < 3 ? "1px solid var(--base-color-gray-200, #E3E3E0)" : "none", background: i % 2 === 0 ? "white" : "var(--base-color-gray-75)", alignItems: "start" }}>

## Badge conventions

Nav badges communicate unread counts or pending items. They're a demand on the user's attention — use them sparingly.

<div>

  {[

## Icon usage

Every nav item has an icon. The icon must communicate the destination independently of the label — users who collapse the sidebar should still navigate correctly.

<div style={{ border: "1px solid var(--base-color-gray-200, #E3E3E0)", borderRadius: 10, overflow: "hidden", marginBottom: 48 }}>
  <div style={{ display: "grid", gridTemplateColumns: "1fr 1fr 1fr", background: "var(--base-color-gray-100, #EEEEEC)", padding: "10px 20px", borderBottom: "1px solid var(--base-color-gray-200, #E3E3E0)" }}>
    {["Destination", "Correct icon", "Why"].map(h => <span key={h} style={{ fontSize: 11, fontWeight: 600, color: "#909090", textTransform: "uppercase", letterSpacing: "0.06em" }}>{h}</span>)}
  </div>
  {[
    { dest: "Overview / Home", icon: "LayoutDashboard", why: "Unambiguous — grid of panels = dashboard." },
    { dest: "Inbox / Messages", icon: "Inbox or MessageSquare", why: "Universal inbox metaphor." },
    { dest: "Team / People", icon: "Users", why: "Two people silhouette = people list." },
    { dest: "Settings", icon: "Settings", why: "Gear = settings. Never use Cog or Sliders here." },
    { dest: "Reports / Analytics", icon: "BarChart2", why: "Bar chart = data/reports. LineChart also acceptable." },
    { dest: "Notifications", icon: "Bell", why: "Bell = notification. Universal." },
    { dest: "Documents / Files", icon: "FileText", why: "Page with lines = document." },
    { dest: "Billing", icon: "CreditCard", why: "Card = payment. Never use DollarSign." },
  ].map(({ dest, icon, why }, i) => (
    <div key={dest} style={{ display: "grid", gridTemplateColumns: "1fr 1fr 1fr", padding: "10px 20px", borderBottom: i < 7 ? "1px solid var(--base-color-gray-200, #E3E3E0)" : "none", background: i % 2 === 0 ? "white" : "var(--base-color-gray-75)" }}>
      <span style={{ fontSize: 13, color: "#222" }}>{dest}</span>
      <span style={{ fontFamily: "monospace", fontSize: 11, color: "#1518A6" }}>{icon}</span>
      <span style={{ fontSize: 12, color: "#909090", lineHeight: 1.5 }}>{why}</span>
    </div>
  ))}
</div>

## Secondary navigation — Tabs

When a destination has sub-sections, use **Tabs** inside the page content — not nested items in the sidebar. Sidebar nesting adds cognitive load and breaks the single-level mental model.
